# known_safe_domains.txt
#
# One registrable domain per line. Lines starting with # are comments/blank
# lines are ignored. This file is the single source of truth, used by:
#   - train_model.py (adds these as extra root-domain safe training examples)
#   - known_safe_domains.py -> app.py (runtime risk-score dampening)
#
# To fix a false positive on a specific legitimate site: just add its
# registrable domain (e.g. "example.com", not "www.example.com" or
# "sub.example.com/path") as a new line below. No code changes needed.
# Runtime effect is immediate (no retrain required); to also improve the
# ML model itself on root-domain URLs generally, retrain after adding.
